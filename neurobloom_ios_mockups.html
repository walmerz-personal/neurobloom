♿ Accessibility Features Built In

  • Large Touch Targets: All buttons and interactive elements are minimum 56x56px (exceeds Apple's 44x44 recommendation)
  • High Contrast: Text meets WCAG AAA standards (7:1 contrast ratio)
  • Large Text: Minimum 17px body text, 18-19px for important actions, 28-34px for headings
  • Clear Hierarchy: Visual weight guides attention naturally
  • Voice Input: Prominent microphone button in chat for users with typing difficulties
  • Simple Language: All copy written at 6th grade reading level
  • Emoji Support: For users with aphasia, emojis supplement text
  • One-Handed Use: All navigation reachable with thumb, no gestures requiring two hands
  • Generous Spacing: 16-24px between interactive elements prevents mis-taps
  • Error Forgiveness: No destructive actions without confirmation

💬 Lilly's Visual Identity

  • Avatar: Warm coral/orange gradient with 🌸 blossom emoji (friendly, approachable)
  • Chat Bubbles: Light gray background (#F0F0F3) to distinguish from user's blue bubbles
  • Tone: Messages use warm, conversational language without exclamation overload
  • Response Style: Validates feelings first, then provides guidance

🎯 Typography Scale

  • Large Title: 34px Bold (Navigation headers)
  • Title 1: 28-32px Bold (Welcome cards, section titles)
  • Title 2: 22-24px Semibold (Card headers)
  • Title 3: 19-21px Semibold (List items, exercise titles)
  • Body: 17px Regular (Chat messages, descriptions)
  • Caption: 13-15px Regular (Metadata, labels)
  • Font: SF Pro Display (iOS system font) for native feel

📐 Layout Principles

  • Safe Margins: 20px horizontal padding on all screens
  • Card Spacing: 12-16px between cards for clear separation
  • Border Radius: 16-20px for cards (friendly, modern), 14px for buttons
  • Shadows: Subtle (0 2px 8px rgba(0,0,0,0.06)) to create depth without distraction
  • Tab Bar: Always visible for easy navigation (no hamburger menus)

🎭 Emotional Design

  • Encouraging Not Punishing: Progress bars show "you're close!" not "you failed"
  • Celebration Moments: Gradient welcome cards highlight wins
  • Gentle Reminders: No aggressive red alerts or guilt-inducing copy
  • Human Touch: Emoji used purposefully to add warmth (🌸 for Lilly, 🌟 for achievements)
  • Calm Palette: Blues convey trust and calm, coral adds warmth without aggression

📱 iOS-Specific Considerations

  • Status Bar: Light content on dark notch area
  • Safe Area: Content respects notch and home indicator
  • Tab Bar: 83px height includes safe area for home indicator
  • Navigation: Large title style for familiarity
  • Gestures: Swipe back to previous screen (iOS standard)
  • Haptics: Gentle feedback on button taps (to be implemented)
  • VoiceOver Ready: All elements have descriptive labels

🔧 Implementation Notes for Claude Code

  • SwiftUI: Use SwiftUI for declarative UI that's easier to maintain
  • Accessibility Modifiers: .accessibilityLabel(), .accessibilityHint(), .accessibilityValue() on all elements
  • Dynamic Type: Support text size adjustments (some users need even larger text)
  • Color Scheme: Support both light and dark mode
  • SF Symbols: Use for icons where appropriate (consistent with iOS)
  • Haptic Feedback: Use UIImpactFeedbackGenerator for button presses
  • Data Persistence: UserDefaults for simple settings, Core Data or SwiftData for progress tracking
  • Offline First: Cache exercise videos locally, sync progress when online
